# M3 照応詞チェック — テストケース仕様

`check_zenshou()` の動作を確認するためのケース集。
GUI またはCLIで入力し、期待するエラー・OKと一致することを手動で確認する。

---

## ケース 1: 多項従属クレームの前記照応（基本）

**目的**: 「請求項1又は2に記載の」形式で、一部の親にしか存在しない用語を「前記」で参照した場合にエラーになること。

```
【請求項１】
人参を有する料理。
【請求項２】
前記人参と茄子とを有する請求項１に記載の料理。
【請求項３】
前記人参と前記茄子と南瓜とを有する請求項１又は２に記載の料理。
```

| 請求項 | 用語 | 期待結果 | 理由 |
|--------|------|----------|------|
| 2 | 前記人参 | OK | 請求項1に「人参」あり |
| 3 | 前記人参 | OK | 請求項1・2どちら経由でも「人参」はスコープ内 |
| 3 | 前記茄子 | **ERROR** | 請求項1に従属した場合「茄子」がスコープ外 |

---

## ケース 2: 範囲指定の多項従属（「から〜のいずれか」）

**目的**: 「請求項1から3のいずれか1項」形式でも同様に検出されること。

```
【請求項１】
人参を有する料理。
【請求項２】
前記人参と茄子とを有する請求項１に記載の料理。
【請求項３】
前記人参と前記茄子と南瓜とを有する請求項２に記載の料理。
【請求項４】
前記人参と前記茄子と前記南瓜と玉葱とを有する請求項１から３のいずれか１項に記載の料理。
```

| 請求項 | 用語 | 期待結果 | 理由 |
|--------|------|----------|------|
| 4 | 前記人参 | OK | 請求項1・2・3どれ経由でも「人参」はスコープ内（請求項1に定義） |
| 4 | 前記茄子 | **ERROR** | 請求項1に従属した場合「茄子」がスコープ外 |
| 4 | 前記南瓜 | **ERROR** | 請求項1・2に従属した場合「南瓜」がスコープ外 |

---

## ケース 3: 「乃至」による範囲指定

**目的**: 「請求項1乃至4の」形式でも同様に検出されること（パーサーのカバレッジ確認）。

```
【請求項１】
人参を有する料理。
【請求項２】
前記人参と茄子とを有する請求項１に記載の料理。
【請求項３】
前記人参と前記茄子と南瓜とを有する請求項２に記載の料理。
【請求項４】
前記人参と前記茄子と前記南瓜と玉葱とを有する請求項３に記載の料理。
【請求項５】
前記人参と前記茄子と前記南瓜と前記玉葱とを有する請求項１乃至４の料理。
```

| 請求項 | 用語 | 期待結果 | 理由 |
|--------|------|----------|------|
| 5 | 前記人参 | OK | 全親経由で「人参」はスコープ内 |
| 5 | 前記茄子 | **ERROR** | 請求項1に従属した場合「茄子」がスコープ外 |
| 5 | 前記南瓜 | **ERROR** | 請求項1・2に従属した場合「南瓜」がスコープ外 |
| 5 | 前記玉葱 | **ERROR** | 請求項1・2・3に従属した場合「玉葱」がスコープ外 |

---

## ケース 4: 単項従属（回帰確認）

**目的**: 通常の単項従属で既存の動作が壊れていないこと。

```
【請求項１】
人参を有する料理。
【請求項２】
前記人参と茄子とを有する請求項１に記載の料理。
【請求項３】
前記人参と前記茄子と南瓜とを有する請求項２に記載の料理。
```

| 請求項 | 用語 | 期待結果 | 理由 |
|--------|------|----------|------|
| 2 | 前記人参 | OK | 請求項1に「人参」あり |
| 3 | 前記人参 | OK | 祖先（請求項1）に「人参」あり |
| 3 | 前記茄子 | OK | 祖先（請求項2）に「茄子」あり |

---

## ケース 5: 独立クレームへの前記（エラー）

**目的**: 従属元のない独立クレームで「前記」を使った場合のエラー検出。

```
【請求項１】
人参を有する料理。
【請求項２】
前記人参と茄子とを有する料理。
```

| 請求項 | 用語 | 期待結果 | 理由 |
|--------|------|----------|------|
| 2 | 前記人参 | **ERROR** | 請求項2は独立クレームのため祖先スコープなし |
